Here Comes Foxy

Foxycat will be a fresh start. Here are a few things I'm going to do differently.

  • Everything but important backups will be deleted and stricter control on what's stored.
  • Run a git server for local repositories.
  • Reduce power consumption.
  • Utilize virtualization or containerization as much as possible.
  • Learn another Linux environment.

What was wrong with Redcat?

When Redcat was created it fulfilled the roles it was created for. However that was 5 years ago, I use my computers differently. I've Lost interest in some hobbies and gained interest in others.ost interest in some hobbies and gained interest in others.

Redcat had two goals.

  1. Give me a place to learn about and enjoy Linux.
  2. Store files for siblings and myself. Mainly to watch pirated movies and tv shows.

Both goals are now irrelevant due to the passage of time. I've enjoyed linux so much that all my computers run it now. I've lost interest in watching movies and TV shows, I've also moved out and no longer have siblings on the same local network. Australian internet has gotten slightly faster therefore the offline storage of movies and tv-shows is no-longer important.

Redcat was running a poorly configured Ubuntu server that often required fixing. Redcat also had plenty of redundant storage totaling a usable capacity of 8tb. Because of the large storage capacity, it's filled up with useless junk. So much it's too time consuming to filter out.
